Process Monitor

Intermediate

Use Python to list running processes, inspect their memory footprints, and identify which programs consume the most resources.

process_monitor.py 
import psutil


def top_processes(limit=10):
    procs = []
    for p in psutil.process_iter(['pid', 'name', 'cpu_percent']):
        try:
            info = p.info
            info['mem_mb'] = p.memory_info().rss / (1024 * 1024)
            procs.append(info)
        except Exception:
            continue

    procs.sort(key=lambda item: item['mem_mb'], reverse=True)
    return procs[:limit]

for i, proc in enumerate(top_processes(), 1):
    print(f"{i}. PID:{proc['pid']:6} | {proc['mem_mb']:6.0f}MB | {proc['name']}")

Why it matters

  1. It exposes the idea of process scheduling and resource usage in a very tangible way.
  2. Memory is measured in real numbers, helping you connect theory to actual system behavior.
  3. The script can be expanded into a monitoring dashboard later.

Disk Usage Reporter

Beginner

Walk a directory tree and report which folders consume the most space so you can understand file system management better.

disk_usage.py 
import os


def folder_sizes(start_path):
    sizes = []
    for root, dirs, files in os.walk(start_path):
        total = 0
        for name in files:
            path = os.path.join(root, name)
            if os.path.isfile(path):
                total += os.path.getsize(path)
        sizes.append((root, total))
    return sizes

for path, size in folder_sizes('.')[:10]:
    print(f'{size:10} bytes  {path}')

Why it matters

  1. It helps you see how filesystems store and organize information.
  2. Walking directories reinforces how operating systems manage access paths.
  3. You can turn it into a cleanup assistant or storage analyzer later.
