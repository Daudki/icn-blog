System Hardware Profiler

Beginner

Use Python to inspect processor details, core count, memory size, and disk information on the machine you are using.

system_profiler.py 
import platform
import psutil


def system_profiler():
    print('CPU:', platform.processor())
    print('Physical cores:', psutil.cpu_count(logical=False))
    print('Logical cores:', psutil.cpu_count(logical=True))

    mem = psutil.virtual_memory()
    print('RAM:', round(mem.total / (1024 ** 3), 2), 'GB')


system_profiler()

Why it matters

  1. It shows how the CPU and memory are exposed through software.
  2. Core counts and memory size connect hardware concepts to measurable values.
  3. The script can become a starter dashboard for system analysis later.

Memory Hierarchy Explorer

Beginner

Compare the roles of registers, cache, RAM, and storage using a simple model that helps you reason about speed and capacity.

memory_hierarchy.py 
hierarchy = [
    ("Registers", "Fastest, smallest, closest to CPU"),
    ("Cache", "Very fast, small, stores frequently used data"),
    ("RAM", "Main working memory for active programs"),
    ("Storage", "Large, slower, persistent data")
]

for level, description in hierarchy:
    print(f"{level}: {description}")

Why it matters

  1. It makes the memory hierarchy easier to remember.
  2. You can see that performance and size trade off across levels.
  3. This becomes a foundation for understanding how programs execute efficiently.
